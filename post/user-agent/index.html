User-Agent

2018.03.03

Um user-agent é um cabeçalho de requisição enviado do cliente (navegador por exemplo) para o servidor( por exemplo web server) tendo como conteúdo algumas informações como: qual o sistema operacional e sua versão que esta sendo usada pelo cliente, IP, qual o tipo de navegador e sua versão . Uma requisição é um pedido, uma solicitação de algo para o servidor, que pode ser uma pagina web, um video ou simplesmente um pacote com alguns bits para ver a rota feita pelos pacotes ate a localização física do servidor.

    O que define quem é cliente e quem é servidor?
  • O que determina isso é o tipo de mensagem HTTP utilizada por cada um. Se request (requisição): cliente, se response (resposta): servidor.
O que acontece?
  • Cliente se conecta ao servidor usando o IP e a porta do tal.
  • O cliente faz um request via protocolo HTTP
  • O servirdor retorna com o HTTP response o resultado processado com infos. como o tipo de servidor, tipo de arquivo retornado, se a conexão é keep-alive (conexao persistente) e um codigo que informa o estado atual do recurso, aqui estão alguns :
    • 200 OK: Recurso encontrado com sucesso;
    • 301 Moved Permanetly: Redirecionamento para outra URL;
    • 302 Found: Recurso temporariamente em outra URL (Pagina de Login, login feito redirecionamento para a pagina principal);
    • 403 Forbidden: Falta privilégios suficientes;
    • 404 Not Found: o servidor não encontrou o recurso (pagina web por exemplo) buscado;
    • 500 Internal Server Error: O servidor nao suporta essa funcionalidade), versão do servidor, e o tipo de conteúdo retornado.
    • Qualquer ferramenta, comando ou script que voce use para fazer uma requisição ao servidor ira com alguma informação e o mínimo que sera enviado sera o tipo de aplicação e sua versão. Isso auxilia o http response retornar os arquivos corretos para a sua versão, evitando bugs ou quaisquer outros tipos de coisas indesejadas.
